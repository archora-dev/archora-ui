# Archora UI Plan

## Current Focus

Build portfolio-grade component depth without AI-themed examples or positioning.

## Active Phase: ArchDataTable Depth Pass

- Strengthen `ArchDataTable` as the first showcase component.
- Keep the existing public API compatible where possible.
- Add practical data-grid behavior before adding more component breadth.

## Scope

- Column visibility through `visibleColumnKeys` and `hidden` column metadata.
- Column sizing through `width` and `minWidth`.
- Built-in pagination through `page` and `pageSize`.
- Error, loading, and empty states with override slots.
- Custom header slots and row action slots.
- Compact density and sticky header support.
- Realistic VitePress examples for portfolio evaluation.

## Deferred

- Virtualized DataTable rows.
- Server-side pagination contracts.
- Column visibility menu UI.
- Resizable or draggable columns.
